# Phase 3 — 시나리오

## 여덟 가지 사용 시나리오

---

## 시나리오 1: 아침 출근 전, 오늘 예산 확인 (철수)

**Context**: 평일 아침 7시, 출근 준비 중

**Flow**:
1. 침대에서 앱 열기
2. 홈 화면에서 "오늘 예산" 섹션 확인
   - 식비 목표: 40,000 / 사용: 0 (0%)
   - 교통비 목표: 5,000 / 사용: 0 (0%)
   - 일일 전체 목표: 50,000 / 사용: 0
3. 아이 도시락 준비하며 "오늘은 외식 줄이자" 다짐

**Insight**:
- 하루를 시작하기 전에 목표를 리마인드받음
- 어제 초과 카테고리 (커피비)를 기억하고 절약 계획 수립
- **앱 가치**: 절약 의지를 매일 강화

---

## 시나리오 2: 점심 먹고 즉시 기록 (철수)

**Context**: 회사 근처 식당에서 점심 (12:30)

**Flow**:
1. 결제 후 바로 스마트폰 꺼내기
2. 앱 앱 → 거래 기록 (+)
3. 금액 입력: 12,000원
4. 카테고리: 식비 (자동 선택 가능)
5. 설명 (선택): "회사 근처 식당" 입력 → 저장
6. **1초 후**: 홈 화면으로 복귀, 현재 "식비 12,000 / 40,000 (30%)" 시각화

**Insight**:
- 3 탭 이내로 완료 (선택지 최소화)
- 즉시 현황 업데이트 → 심리적 만족감
- "어? 벌써 30% 썼네?" 놀라움 → 절약 동기 자극

**P설계 원칙**: P1(빠른 입력 우선), P3(필드 최소화), P5(한눈에 현황)

---

## 시나리오 3: 칼장값 초과 경고 → 저녁 메뉴 조정 (철수)

**Context**: 오후 3시, 커피 모임 중 (카페)

**Flow**:
1. 아메리카노 3,800원 기록
2. 앱이 알림 표시: "⚠️ 커피값이 4,600원 초과했습니다"
   - 목표: 5,000 (식비 일부)
   - 사용: 9,600 (커피 3잔)
3. 철수: 전화로 아내에게 "오늘 저녁은 간단히 먹자" 제안
4. 저녁에 편의점 김밥(4,000원) 구매 → 기록

**Insight**:
- 목표 초과를 실시간으로 경고 → 행동 개선
- 아내와 즉시 공유 가능 → 가족 예산 조율
- **앱 가치**: 충동 구매 방지 + 절약 실천

**P설계 원칙**: P2(선택적 목표), P4(모바일 우선), P5(한눈에 현황 + 알림)

---

## 시나리오 4: 월말 지출 분석 → 다음 달 목표 재조정 (철수 & 아내)

**Context**: 일요일 저녁 (부부 회의)

**Flow**:
1. 앱 열기 → "7월 통계" 탭
2. 월간 지출액: 2,850,000원 (목표: 3,000,000)
   - 식비: 850,000 (예상: 840,000) ✓ 초과 10,000
   - 교통비: 180,000 (예상: 150,000) ✗ 초과 30,000
   - 쇼핑: 580,000 (예상: 600,000) ✓ 절약 20,000
3. 원형 차트로 카테고리별 비율 시각화
4. 아내와 함께 보며 "교통비 좀 줄여보자" 논의
5. **8월 목표 수정**:
   - 교통비 목표: 5,000 → 4,500원/일로 조정
   - 다른 카테고리는 유지

**Insight**:
- 데이터 기반 가족 재정 회의
- 객관적인 지출 패턴 파악 → 개선점 도출
- **앱 가치**: 투명한 재정 관리 + 목표 기반 개선

**P설계 원칙**: P1(빠른 입력) + P4(월간 통계) = 신뢰할 수 있는 데이터

---

## 시나리오 5: 바쁠 때 5초 만에 기록 (영희)

**Context**: 강의 끝나고 (15:00), 카페에서 아이스 아메리카노 구매

**Flow**:
1. 결제 후 스마트폰 꺼내기 (2초)
2. 앱 열기 → "+" 버튼 (1초)
3. 금액 입력: "4500" (1초)
4. 카테고리 선택: "식비" (1초)
5. 저장 (1초)
6. **돌아가기**: 즉시 강의실로

**Insight**:
- 거래 입력이 방해 없음
- 설명(memo) 필드 없어도 불편하지 않음
- **영희의 필요**: 기록 그 자체가 목적 (분석은 X)

**P설계 원칙**: P1(빠른 입력 우선), P2(필드 최소화 - 필수만)

---

## 시나리오 6: 자기 전, 간단 지출 현황 확인 (영희)

**Context**: 밤 11:30, 침대에 누워

**Flow**:
1. 앱 홈 화면 열기
2. "오늘 지출: 45,500원 / 월 평균: 42,000원" 확인
3. "어? 이번 달은 좀 많이 썼네" 생각
4. 닫기 (추가 분석 X)

**Insight**:
- 간단한 현황만으로 충분
- 디테일한 차트나 분석은 필요 없음
- **영희의 필요**: 간단한 추적과 확인

**P설계 원칙**: P2(필드 최소화), P5(홈에서 한눈에 기본 정보)

---

## 시나리오 7: 친구와 나눠 먹었을 때 입력 (영희)

**Context**: 점심 시간, 친구와 김밥 한 줄 반으로 나눠 먹음 (총 6,000원)

**Flow**:
1. 영희 몫: 3,000원 (가계부에 기록)
2. 앱 열기 → "+" → 3000 → 식비 → 저장
3. (친구와는 따로 정산 X)

**Insight**:
- 입력은 자유도 높음 (어떻게 기록할지 사용자 판단)
- 앱은 금액과 카테고리만 관리
- **영희의 유연성**: "정확한 기록"보다 "빠른 기록"

**P설계 원칙**: P3(필드 최소화), P1(빠른 입력)

---

## 시나리오 8: 첫 설치 후 온보딩 (철수 & 영희의 공통 길)

**Context**: 앱 첫 다운로드 직후

### 8-1. 철수의 경로
1. 앱 열기 → "처음이신가요?"
2. **목표 설정 유도**: "월 목표를 정해주세요" (선택)
   - 월 예산: 3,000,000
   - 카테고리별 일일 목표: 식비 40,000 등
3. 샘플 거래 입력해보기
4. 홈 화면 투어 (목표 진행률 하이라이트)
5. 바로 사용 시작

**Insight**: 철수는 목표부터 시작 → 절약 동기 강화

### 8-2. 영희의 경로
1. 앱 열기 → "처음이신가요?"
2. **목표 설정 스킵 가능** (선택사항)
3. 바로 "거래 입력" 버튼으로 이동
4. 샘플 거래 3개 입력해보기
5. 홈 화면 (간단 통계만 표시)
6. 바로 사용 시작

**Insight**: 영희는 복잡한 설정 없이 빠르게 시작 → 낮은 진입장벽

**P설계 원칙**: P3(목표는 선택적), P1(빠른 입력), P2(필드 최소화)

## 핵심 인사이트

### 1. 입력의 시간성
- **철수**: 거래 직후 즉시 입력 (심리적 부담 낮음)
- **영희**: 바쁠 때도 5초 만에 입력 (방해 없음)
→ **설계**: 3 탭 이내, 필드 최소화 필수

### 2. 목표 기능의 역할
- **철수**: 절약 동기 자극 + 실시간 경고 → 행동 변화
- **영희**: 무관 (선택적)
→ **설계**: 목표는 선택적이지만, 설정하면 강력한 기능

### 3. 통계와 분석
- **철수**: 월말 분석 → 다음 달 목표 조정 (중요)
- **영희**: 간단 현황만 (깊이 있는 분석 X)
→ **설계**: 기본 차트는 제공, 고급 분석은 v2

### 4. 모바일의 중요성
- 모든 시나리오가 스마트폰 기반
- 외출 중 즉시 입력 필요
→ **설계**: 모바일 우선, 데스크톱은 보조

### 5. 투명성과 신뢰
- 철수가 가족과 공유하고 함께 분석 → 신뢰 구축
- 데이터 기반의 재정 의사결정
→ **설계**: 명확한 수치와 시각화
