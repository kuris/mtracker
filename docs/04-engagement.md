# Phase 4 — 차별화·성취감

## 선택된 차별화 기능

사용자가 선택한 3가지 기능:
1. **날씨 느낌 표시** (높음/낮음)
2. **절약 성과 실시간 반영**
3. **월간 절약액 표시**

---

## 구체적 차별화 기능 (E1 ~ E3)

### E1. 일일 지출 '날씨' 시각화

**목적**: 오늘 지출이 많았는지 적었는지 한눈에 느끼게 하기

**구현**:
- 홈 화면 상단에 **오늘 기준선** 표시
  - 예: 목표 50,000원 기준
  - 지출 20,000원 (40%) → 냉냉 절약했어요 (파란색)
  - 지출 52,000원 (104%) → 조금 많이 썼어요 (주황색)
  - 지출 70,000원 (140%) → 많이 썼어요 (빨간색)

**기술**:
- 일일 목표 vs 현재 지출액으로 계산
- 이모지 + 문구 + 색상으로 감정 표현
- 진행률 바와 함께 표시

**심리 효과**:
- 추상적 수치 (42%) → 구체적 느낌 (절약했어요)
- 매일 작은 성취감 누적
- 초과 시 즉시 경고

---

### E2. 절약 성과 실시간 반영 (Daily Progress)

**목적**: 목표 달성을 향한 작은 승리 체험

**구현**:

#### 2-1. 금액 기반 목표 (Amount-based)
- 카테고리별로 목표 대비 절약액 실시간 표시
  `
  식비 목표: 40,000원
  현재 사용: 32,000원
  
  절약액: 8,000원 (20% 남음)
  "아직도 8,000원 남았어요!"
  `

#### 2-2. 횟수 기반 목표 (Count-based) - 사용자 요청 추가
- 특정 카테고리를 횟수로 제한
  - 예: 커피 3잔 목표, 쇼핑 주 2회 목표
  - 사용자가 거래 입력할 때마다 카운트 감소

**입력 시 즉시 피드백**:
  `
  1. 커피 3,800원 입력
  → 메시지: "오늘 커피 3잔 중 2잔 남았어요! ☕☕"
  
  2. 쇼핑 50,000원 입력
  → 메시지: "이번 주 쇼핑 2회 중 1회 남았어요! 🛍️"
  
  3. 초과 입력
  → 경고: "어? 커피를 3잔 이상 마셨네요! ⚠️"
  `

**카테고리별 카운트 시각화**:
  `
  커피: ☕ ☕ ⭕ (2잔 남음)
  쇼핑: 🛍️ ⭕ (1회 남음)
  `

**목표 달성 메시지**:
  `
  유지: "오늘도 커피 3잔 목표 달성! 👍"
  초과: "어? 커피를 3잔 이상 마셨네요!" (경고)
  `

#### 2-3. 스트릭 (Streak) 기능
- "연속 3일 목표 달성!" (작은 배지)
- 주간 목표 달성일 카운트 표시

**기술**:
- 실시간 계산 (Supabase Realtime)
- 금액 기반: 진행률 바 (0% ~ 100%)
- 횟수 기반: 카운트 바 (●●○○ 형태) + 남은 횟수 숫자
- 거래 입력 직후 애니메이션 효과

**심리 효과**:
- 금액: "아직도 5,000원 쓸 수 있어" → 긍정적 동기
- 횟수: "2잔 남았어!" → 구체적이고 명확한 제한감
- 매 거래마다 즉시 피드백 → 습관 형성 가속
- 작은 성공의 축적

---

### E3. 월간 절약액 상세 표시

**목적**: 한 달간의 노력을 구체적 수치로 검증

**구현**:
- 월 화면 상단에 절약액 카드
  `
  이번 달 vs 지난달
  
  목표: 3,000,000원
  사용: 2,750,000원
  절약액: 250,000원
  
  지난달 대비: -150,000원 (6% 감소)
  `

- 카테고리별 절약액
  - 식비: 50,000원 절약 (체크마크)
  - 교통비: 20,000원 초과 (경고)
  - 쇼핑: 80,000원 절약 (체크마크)

- 연월비 차트
  - 지난 3개월 절약액 트렌드 (막대 그래프)

**기술**:
- 월별 집계 쿼리 (지난달 vs 이번달)
- 동적 계산 (선택된 목표 기준)
- 카테고리별 분류

**심리 효과**:
- "250,000원 절약했다" → 구체적 성과 체감
- 목표 설정과 달성의 연결고리
- 가족 함께 성과 공유 가능 (부부 동기 강화)

## 의도적으로 넣지 않는 것 (Non-Features)

다음은 v1에서 의도적으로 제외한 기능들:

- 게임화 (게임 포인트, 뱃지, 레벨): 과하면 진지한 재정 관리 이미지 손상
- 친구 비교 (랭킹, 경쟁): 부부 가계부에 경쟁은 필요 없음
- 일일 알림/푸시: 일시적 주의 집중, 장기 습관화 어려움
- 자동 카테고리 분류 (AI): v1에서는 수동 입력으로 의도성 강화
- 절약팁 제시: 너무 많은 정보로 인한 피로도

철학: 절약 성과를 명확하게 보여주되, 과도한 자극은 피한다.

## 기술 구현 전략

### 프론트엔드
1. E1 (날씨): React 컴포넌트로 진행률 기반 동적 색상/이모지
2. E2 (금액/횟수): 카운트 바 + 진행률 바 컴포넌트
3. E3 (월간): 월별 집계 쿼리 결과를 차트 라이브러리로 렌더링

### 백엔드 (Supabase)
1. E1: daily_goals vs transactions (당일) 실시간 계산
2. E2:
   - 금액: amount 합계 계산
   - 횟수: 카테고리별 transaction count 계산
3. E3: transactions를 월별로 집계

### 데이터 모델 확장
- daily_goals 테이블에 goal_type 추가 ('amount' | 'count')
- count 목표일 때: target_count, current_count 필드

### 성능
- E1, E2: 실시간 (< 500ms, Realtime 이용)
- E3: 캐시 (월 1회 새로고침)

## 온보딩 (Onboarding) - 습관 트리거 설정

**목적**: 첫 가입 시 사용자의 습관(커피, 술, 담배 등)을 파악해 처음부터 횟수 목표를 설정하게 하기

**구현**:

### 가입 후 온보딩 플로우
1. "당신의 습관을 알려주세요" 화면
2. 선택지 제시 (다중 선택 가능):
   ```
   ☕ 커피를 자주 마시시나요?
   → 예면: "하루 몇 잔?"
      - 1잔
      - 2잔
      - 3잔
      - 4잔 이상
   
   🍺 술을 자주 마시시나요?
   → 예면: "주 몇 회?"
      - 안 마심
      - 주 1회
      - 주 2-3회
      - 주 4회 이상
   
   🚬 담배를 피우시나요?
   → 예면: "하루 몇 개비?"
      - 안 픔
      - 적게 (5개비)
      - 중간 (10개비)
      - 많이 (15개비 이상)
   ```

3. 선택 후 **자동으로 일일 횟수 목표 설정**
   ```
   "좋아요! 다음과 같이 설정했어요"
   
   ☕ 커피: 하루 3잔 목표
   🍺 술: 주 2회 목표
   ```

4. "시작하기" 버튼 → 앱 진입

### 데이터 모델 확장
```sql
user_habits {
  id: UUID (PK)
  user_id: UUID (FK → users.id)
  category: string ('coffee' | 'alcohol' | 'cigarette' | ...)
  goal_type: 'count' (횟수 기반 필수)
  target_count: integer (하루/주 목표)
  frequency: 'daily' | 'weekly'
  created_at: timestamp
}
```

### 심리 효과
- 첫 가입 시점에 목표 설정 → 초기 동기 강화
- 앱 진입 직후 바로 "2잔 남았어요" 메시지 → 즉시 몰입
- 사용자가 자신의 습관을 인정하고 제한하려는 의지 표현
- "내가 원했던 습관 관리"를 바로 시작 가능
