# Phase 2 — 페르소나

## 주 페르소나: 김철수 (절약 실천하는 부부)

### 프로필
- **이름**: 김철수
- **연령/직업**: 35세 직장인
- **가족 상황**: 아내(이지현, 32세)와 함께 생활하는 2인 가구
- **수입**: 연소득 6,000만원대 (부부 합산)
- **기술 친화도**: 중상 (앱 사용에 거부감 없음)

### 목표 & 동기
- **주요 목표**: 가족 생활비를 효율적으로 관리하고 월별 절약액 달성
- **동기**: 5년 내 주택 구매 자금 모으기, 과소비 억제
- **성공의 정의**: "월 300만원 이상 절약 달성"

### 불만 & 우통점
- **기존 도구의 문제**:
  - 입력이 복잡해서 거래 직후 기록하지 않고 나중에 몰아 입력 → 실수 증가
  - 통계 보기까지 여러 단계 필요 → 귀찮음
  - 아내와 데이터 공유 안 됨 (각자 관리)
- **욕구**:
  - 1초 내에 거래 기록 가능
  - 하루 예산을 정해서 초과되지 않도록 경고받기
  - 지출 현황을 한눈에 보기

### 행동 패턴
- **언제 사용?**: 아침 9시~저녁 6시 (출근 중/점심 때/퇴근 후)
- **거래 빈도**: 하루 4~5번 (아침 커피, 점심, 저녁 식사, 편의점 등)
- **분석 시점**: 주말 저녁, 월말 (가족 회의)
- **기기**: 업무용 스마트폰 + 가끔 PC

### 설정할 목표 예시
`
월 예산: 3,000,000원
카테고리별 일일 목표:
- 식비: 40,000원/일
- 교통비: 5,000원/일
- 쇼핑: 20,000원/일
`

## 대조 페르소나: 이영희 (간편하게 관리하는 1인)

### 프로필
- **이름**: 이영희
- **연령/직업**: 26세 대학생 (아르바이트)
- **가족 상황**: 혼자 생활
- **수입**: 월 150만원 (아르바이트)
- **기술 친화도**: 높음 (MZ세대)

### 목표 & 동기
- **주요 목표**: 용돈 관리, 불필요한 지출 줄이기
- **동기**: 저축 목표 100만원/월 달성
- **성공의 정의**: "매일 간단하게 기록하고 월 패턴 파악"

### 불만 & 우통점
- **기존 도구의 문제**:
  - 너무 많은 기능이 불필요함
  - 간단한 기록에 5번 이상의 탭/클릭 필요
  - 부부 기능/복잡한 통계는 관심 없음
- **욕구**:
  - 진입 장벽이 낮음 (최소한의 필드만)
  - 거래 입력 후 바로 진행 상황 보기
  - 카테고리는 간단하게 (식비/쇼핑/기타 정도)

### 행동 패턴
- **언제 사용?**: 점심/저녁 시간, 자기 전
- **거래 빈도**: 하루 2~3번 (주로 식비, 쇼핑)
- **분석 시점**: 주 1회 (월요일) 지출 현황 확인
- **기기**: 스마트폰 100%

### 설정할 목표 예시
`
월 예산: 1,500,000원
카테고리별 일일 목표:
- 식비: 25,000원/일
- 쇼핑: 10,000원/일
`

## 두 페르소나의 긴장 축과 설계 영향

### 긴장 축 비교표

| 축 | 김철수 (주 페르소나) | 이영희 (대조 페르소나) | 설계 반영 |
|---|---|---|---|
| **가구 구성** | 부부 (2인) | 1인 | 공유 기능은 optional, 단독 사용이 기본 |
| **입력 빈도** | 하루 4~5회 | 하루 2~3회 | 빠른 입력이 최우선 (1초 타겟) |
| **거래 규모** | 소액 多 (5K~50K) | 소액 多 (3K~30K) | 빠른 입력 지원 |
| **목표 활용** | 높음 (절약 실천) | 낮음 (참고만) | 목표 설정을 강제하지 않기 |
| **통계 관심도** | 높음 (월말 분석) | 낮음 (간단 확인) | 기본 통계만 제공, 깊이 있는 분석은 v2 |
| **카테고리** | 8개 (세분화) | 3~4개 (간단함) | 기본 카테고리는 심플, 커스텀 추가 가능 (v2) |
| **기기** | 스마트폰 + PC | 스마트폰 만 | 모바일 우선 (그러나 데스크톱도 지원) |
| **입력 난이도** | 중간 (세부 정보) | 최소 (필수만) | 필수 필드: 금액, 카테고리만. 설명은 optional |

### 설계 원칙 (P1~P5)

| 원칙 | 설명 | 이유 |
|---|---|---|
| **P1. 빠른 입력 우선** | 3 탭 이내로 거래 기록 완료 | 김철수가 매일 여러 번 입력하는데 복잡하면 기록 안 함 |
| **P2. 필드 최소화** | 금액, 카테고리, (선택) 설명 | 이영희처럼 간단함을 원하는 사용자도 있음 |
| **P3. 목표는 선택적** | 목표 설정 안 해도 앱 사용 가능 | 이영희는 목표 관심 없지만, 김철수는 필수 |
| **P4. 모바일 우선** | 스마트폰에서 모든 작업 가능 | 두 페르소나 모두 스마트폰 사용 비율 높음 |
| **P5. 한눈에 현황** | 홈 화면에서 오늘 지출/목표 진행률 표시 | 여러 번의 탭 없이 바로 확인 가능해야 함 |

## 페르소나 간 차이 수용 전략

### 1. 기본값으로 두 페르소나 모두 만족
- **필수 기능**: 입력 + 간단 통계 (김철수 + 이영희 모두 필요)
- **선택 기능**: 목표 설정, 카테고리 커스텀 (김철수 우선, 이영희는 스킵 가능)

### 2. 진입 경로 차이
- **김철수**: 앱 설치 → 목표 설정 → 거래 입력 (목표부터 시작)
- **이영희**: 앱 설치 → 바로 거래 입력 (목표 스킵)

### 3. 화면 레이아웃
- **홈 화면**: 오늘 지출액 + 목표 진행률 (있으면)
- **거래 입력**: 금액 + 카테고리 + 설명(선택)
- **통계**: 월간 요약 차트 (김철수 필수, 이영희는 가끔)

## 요약

**주 페르소나 (김철수)**는 부부 예산을 효율적으로 관리하면서 빠르게 입력하고, 목표 기반으로 절약을 실천하는 사용자입니다.

**대조 페르소나 (이영희)**는 개인 지출을 간단하게 추적하면서 복잡한 기능 없이 필수만 사용하는 젊은 사용자입니다.

→ **설계 핵심**: 빠른 입력과 필드 최소화를 유지하면서도, 목표 기능(김철수)과 통계(김철수)를 충분히 지원해야 합니다.
