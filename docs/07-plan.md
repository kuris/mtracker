## 마일스톤별 세부 계획

### M1: 초기 설정 + 기본 레이아웃 (1시간)

**목표**: "앱이 로컬에서 떠도, UI 기본 구조가 완성됨"

**할 일**:
- [ ] Next.js 프로젝트 초기화 (pnpm create next-app)
- [ ] TypeScript, Tailwind CSS 설정
- [ ] 기본 폴더 구조 생성 (app, components, store, lib, types)
- [ ] Zustand 설정 및 기본 스토어 생성 (authStore, uiStore)
- [ ] 기본 레이아웃 컴포넌트 (Header, Navigation)
- [ ] 글래스모피즘 CSS 변수 설정 (배경 그래디언트, 색상)
- [ ] 홈 페이지 기본 UI 스켈레톤 (목표 카드, 카테고리 카드 자리만)

**검증**:
- [ ] npm run dev 실행 → localhost:3000 정상 로드
- [ ] 기본 레이아웃 시각적 확인 (글래스모피즘 배경)
- [ ] 모바일 반응형 확인

### M2: 인증 (회원가입/로그인) (1.5시간)

**목표**: "사용자가 이메일/비밀번호로 가입하고 로그인 가능"

**할 일**:
- [ ] Supabase 프로젝트 생성 + 환경 변수 설정
- [ ] Supabase Auth 활성화
- [ ] signup/page.tsx 작성 (회원가입 폼)
- [ ] login/page.tsx 작성 (로그인 폼)
- [ ] React Hook Form + Zod로 폼 검증
- [ ] AuthStore 구현 (login, signup, logout, setUser)
- [ ] middleware.ts 작성 (인증 확인, 리다이렉트)
- [ ] 기본 accounts 테이블에 사용자 기본 가계부 자동 생성

**검증**:
- [ ] 회원가입 → 이메일 저장 됨
- [ ] 로그인 → JWT 토큰 발급
- [ ] 로그인 후 홈 페이지 접근 가능
- [ ] 로그아웃 → 로그인 페이지로 리다이렉트
- [ ] 미인증 사용자 → 로그인 페이지로 강제 리다이렉트

### M4: 목표 기능 + 카운트 (1시간)

**목표**: "사용자가 일일 목표를 설정하고 카운트 진행도를 볼 수 있음"

**할 일**:
- [ ] 온보딩 흐름 (첫 가입 시 습관 물어보기)
  - "하루에 커피 몇 잔?"
  - "술은 주 몇 회?"
- [ ] DailyGoal 타입 수정 (금액 + 횟수 목표 동시 지원)
- [ ] GoalStore 구현 (setGoal, updateGoal)
- [ ] 카테고리별 카운트 로직 (횟수 기반)
- [ ] 카테고리 카드에 카운트 표시 (☕☕⭕)
- [ ] 남은 횟수 텍스트 ("2잔 남았어요!")

**검증**:
- [ ] 회원가입 시 온보딩 화면 나옴
- [ ] 목표 저장 → GoalStore에 반영
- [ ] 거래 입력 시 카운트 감소 (즉시 업데이트)
- [ ] 횟수 목표 초과 시 경고 표시

**검증**:

## 완료 기준

각 마일스톤이 "완료"되려면:

1. **기능 완성**: 할 일 모두 체크 완료
2. **로컬 테스트**: npm run dev에서 정상 작동
3. **타입 검사**: \pnpm tsc\ 에러 없음
4. **린팅**: \pnpm lint\ 에러 없음
5. **시각적 확인**: 디자인 문서와 일치
6. **모바일 테스트**: Chrome DevTools에서 반응형 확인
